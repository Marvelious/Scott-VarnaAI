## Three-Phase Service Delivery

### Phase 1: Initial Assessment (Week 1)

**Objective**: Understand current compliance state and identify gaps

**Activities**:
1. **Day 1-2: Discovery Meeting** (Face-to-face in Varna/remote for others)
   - Business overview and operations
   - Current data handling practices
   - IT infrastructure overview
   - Existing compliance documentation
   - Key stakeholders and decision-makers

2. **Day 3-4: Automated Assessment**
   - Run C3 compliance scan (60 seconds)
   - Digital presence review
   - Public-facing compliance audit
   - NIS2 applicability determination
   - GDPR data mapping preliminary

3. **Day 5: Gap Analysis Report**
   - Current compliance score (0-100)
   - Identified gaps by regulation
   - Risk prioritization (Critical/High/Medium/Low)
   - Preliminary remediation estimate

**Deliverables**:
- [ ] Gap Summary Report (Bulgarian language)
- [ ] Compliance Score Dashboard access
- [ ] Initial risk assessment
- [ ] Phase 2 proposal

**Pricing**: Безплатно (Free) - Lead generation / trust building

---

### Phase 2: Remediation Plan (Week 2-4)

**Objective**: Create and execute compliance implementation plan

**Activities**:

#### Week 2: Planning
1. **Detailed Gap Analysis**
   - Full GDPR data inventory
   - NIS2 requirements mapping (if applicable)
   - GoBD financial compliance check
   - BSI C5 cloud assessment (if cloud services used)

2. **Remediation Roadmap**
   - Prioritized action items
   - Timeline with milestones
   - Resource requirements
   - Cost estimates for external services (legal, IT)

3. **Document Preparation**
   - Privacy policy drafts
   - Data processing agreements
   - Cookie consent implementation
   - Employee data handling procedures

#### Week 3-4: Implementation Support
4. **Technical Implementation**
   - Cookie consent banner setup
   - Privacy policy deployment
   - DSAR (Data Subject Access Request) process
   - Data breach notification procedure

5. **Training Materials**
   - Employee awareness training (Bulgarian)
   - Manager compliance responsibilities
   - Incident response procedures

6. **Documentation Package**
   - Complete compliance documentation
   - Audit trail setup
   - Evidence collection procedures

**Deliverables**:
- [ ] Full Compliance Roadmap (timeline + milestones)
- [ ] All required compliance documents (Bulgarian)
- [ ] Technical implementation complete
- [ ] Training materials delivered
- [ ] Initial audit evidence package

**Pricing**:
- **Standard Audit**: 980 лв (~€500) - one-time
- **With Implementation**: 1,960 лв (~€1,000) - one-time

## Appendix B: Compliance Regulation Quick Reference

### GDPR (Общ регламент за защита на данните)
- **Applies to**: All companies processing EU citizen data
- **Key requirements**: Lawful basis, data minimization, DSAR handling, breach notification
- **Penalties**: Up to €20M or 4% global revenue

### NIS2 (Директива за мрежова и информационна сигурност)
- **Applies to**: Essential and important entities (energy, transport, banking, health, digital infrastructure)
- **Key requirements**: Risk management, incident reporting, supply chain security
- **Bulgarian transposition**: Expected 2025
- **Penalties**: Up to €10M or 2% global revenue

### GoBD (Grundsätze zur ordnungsmäßigen Führung und Aufbewahrung)
- **Applies to**: Companies with German business relationships
- **Key requirements**: Digital record keeping, audit trails, data retention
- **Relevance for Bulgaria**: German parent companies or customers

### BSI C5 (Cloud Computing Compliance Criteria Catalogue)
- **Applies to**: Cloud service providers and users
- **Key requirements**: Security controls for cloud services
- **Relevance for Bulgaria**: Companies using German cloud services
