### 2.2 Key Regulatory Drivers for FwChange

#### 2.2.1 BaFin & DORA (Financial Sector)

**BaFin Supervision Scope:**
- **2,700 banks** under supervision[^8]
- **800 financial services institutions**[^8]
- **700+ insurance companies**[^8]

**DORA (Digital Operational Resilience Act) - Effective Jan 17, 2025:**
- **Scope in Germany:** 3,600+ financial entities[^9]
- **Network Security Requirements:**
  - Mandatory vulnerability assessments and network security testing[^9]
  - Threat-Led Penetration Testing (TLPT) for critical entities[^9]
  - **Encryption in use** for sensitive data[^9]
  - Incident reporting: 4-hour initial, 72-hour intermediate, 1-month final reports[^9]
- **Penalties:** Up to 2% of global turnover or €5M for critical ICT providers[^9]

**FwChange Value Proposition for DORA:**
- Automated firewall change audit trails for compliance evidence
- Real-time vulnerability detection in firewall rules
- Integration with ITSM (ServiceNow/Jira) for incident workflow
- Compliance-ready reporting for BaFin audits

**BAIT Transition:**
BaFin repealed ZAIT, VAIT, KAIT in January 2025; BAIT fully repealed December 31, 2026 as DORA takes precedence.[^8] Financial institutions transitioning from BAIT to DORA need firewall change management solutions supporting both frameworks during 2025-2026 transition.

#### 2.2.2 NIS2 Directive (Critical Infrastructure)

**NIS2 Implementation in Germany (Enacted Dec 6, 2025):**
- **Scope Expansion:** 29,500 entities now covered (vs. 4,500 previously)[^10]
- **Sectors:** Digital services, cloud providers, data centers, manufacturing, postal/courier, energy, telecom, transport[^10]
- **Registration Deadline:** January 6, 2026 via BSI portal[^11]

**Security Requirements:**
- **Risk Management Measures:** MFA for admin accounts, network segmentation, ransomware-proof backups[^11]
- **Incident Reporting:** 24-hour initial reporting timeline[^10]
- **Supply Chain Security:** Strengthened controls[^10]
- **Management Training:** Mandatory 4-hour cybersecurity training within 3 years[^11]

**Penalties:**
- **Essential Entities:** €10M or 2% global turnover[^11]
- **Important Entities:** €7M or 1.4% global turnover[^11]
- **Personal Liability:** Management held accountable for non-compliance[^10]

**FwChange Value Proposition for NIS2:**
- Automated network segmentation validation for firewall rules
- Real-time compliance dashboards for BSI reporting
- Multi-vendor firewall orchestration (Palo Alto, Check Point, Cisco, Fortinet)
- Integration with ITSM for 24-hour incident response workflows

#### 2.2.3 TISAX (Automotive Industry)

**TISAX (Trusted Information Security Assessment Exchange):**
- **Framework Basis:** VDA ISA + ISO 27001 Annex A + Privacy controls[^12]
- **Industry Requirement:** BMW, Audi, and major OEMs mandate TISAX for all suppliers[^12]
- **Protection Levels:**
  - **Level 1:** Basic IT security
  - **Level 2:** Comprehensive ISMS (Information Security Management System)
  - **Level 3:** High protection with robust risk management[^12]

**Network Security Controls:**
- Firewall requirements follow ISO 27001/27002 technical controls[^12]
- Governance and oversight for information security
- Physical safeguards and operational resilience[^12]
- Risk management to prevent data breaches[^12]

**FwChange Value Proposition for TISAX:**
- ISO 27001-compliant firewall change workflows
- Audit-ready documentation for TISAX assessments
- Risk scoring for prototype protection and sensitive data flows
- Integration with automotive supplier networks
